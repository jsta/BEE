Graph Database Interface¶

Mid-level interface for managing a graph database with workflows.

Delegates the actual work to an instance of a subclass of the abstract base class GraphDatabaseDriver. By default, this is the Neo4jDriver class.

__hash__()¶

Return the hash value for a task.

__init__(name, command, hints, subworkflow, inputs, outputs)¶

Store a task description.

There are two special tasks: those with the name bee_init and bee_exit.

bee_init should have no dependencies and all head tasks of the user’s workflow should depend on it.

bee_exit should have no dependents and depend on all of the end tasks in the user’s workflow.

Parameters

  • task_id (integer) – the task’s unique ID

  • name (string) – the task name

  • command (list of strings) – the command to run for the task

  • hints (dictionary) – the task hints (optional requirements)

  • subworkflow (string) – an identifier for the subworkflow to which the task belongs

  • inputs (set) – the task inputs

  • outputs (set) – the task outputs
